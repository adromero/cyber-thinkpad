# GitHub Setup Guide

This guide will help you push this project to GitHub for the first time.

## Prerequisites

The project is ready for GitHub! All hardcoded paths have been removed and the repository has been initialized.

## Step 1: Configure Git Identity

Before making your first commit, configure your Git identity:

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

Or set it just for this repository:

```bash
git config user.name "Your Name"
git config user.email "your.email@example.com"
```

## Step 2: Create the Initial Commit

All files are already staged. Create the initial commit:

```bash
git commit -m "Initial commit: ThinkPad Cyberpunk system monitoring and rice

Add cyberpunk-themed system monitoring suite for ThinkPad laptops:
- Battery management with charging threshold control (batctl)
- Thermal monitoring with real-time alerts (thermctl)
- Power profile management (powerctl)
- Cyberpunk dashboard with real-time stats (cyberdash)
- Status bar widgets for i3/polybar (cyberbar)
- Interactive keybindings reference (cyberkeys)
- Workspace launcher for i3 (workspacelaunch, workspaceconfig)

Add cyberpunk desktop rice with configs for:
- i3 window manager
- Polybar status bar
- Rofi launcher
- Kitty terminal
- Picom compositor

Features:
- Pure Python 3 with no external dependencies for monitoring tools
- Unified installation script with optional rice installation
- Dynamic path resolution (no hardcoded paths)
- Cross-distro support (Ubuntu, Debian, Arch, Fedora)
- Comprehensive documentation and contribution guidelines"
```

## Step 3: Create GitHub Repository

1. Go to https://github.com/new
2. Create a new repository named `thinkpad-cyberpunk`
3. **DO NOT** initialize with README, .gitignore, or license (we already have these)
4. Choose visibility (Public or Private)
5. Click "Create repository"

## Step 4: Add Remote and Push

After creating the repository on GitHub, connect your local repository:

```bash
# Add the remote (replace YOUR_USERNAME with your GitHub username)
git remote add origin https://github.com/YOUR_USERNAME/thinkpad-cyberpunk.git

# Push to GitHub
git push -u origin main
```

## Step 5: Update README

After pushing, update the clone URL in README.md:

1. Edit `README.md`
2. Find the line: `git clone https://github.com/YOUR_USERNAME/thinkpad-cyberpunk.git`
3. Replace `YOUR_USERNAME` with your actual GitHub username
4. Commit and push the change:

```bash
git add README.md
git commit -m "Update clone URL in README"
git push
```

## Troubleshooting

**If you get authentication errors:**
- Use SSH instead: `git remote set-url origin git@github.com:YOUR_USERNAME/thinkpad-cyberpunk.git`
- Or configure GitHub CLI: `gh auth login`

**If you need to change the remote URL:**
```bash
git remote set-url origin https://github.com/YOUR_USERNAME/thinkpad-cyberpunk.git
```

**To verify your remote:**
```bash
git remote -v
```
